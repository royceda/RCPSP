IBM ILOG CPLEX Optimizers 12.6.0

This directory includes the CPLEX Optimizers of IBM® ILOG® CPLEX® Optimization Studio 12.6.0, including Concert Technology for C++ and Java users. Please read the CPLEX Optimization Studio 12.6.0 Release Notes in order to take full advantage of the new features in this release and to ensure a smooth transition from prior versions of CPLEX.

Directory Structure

The following subdirectories are contained in the directory where you installed CPLEX Optimization Studio 12.6.0.

In the cplex directory:

  bin/<machine>:
	Executable files and shared libraries, including those listed here.  <ext> varies
	with the operating system. 
	  cplex - CPLEX Interactive Optimizer
	  libcplex1260.<ext> - CPLEX shared library for JNI.
	  libcplex1260remote.<ext> - CPLEX remote object library
	  libcplex1260remotejni.<ext> - JNI bindings for the remote object
	  libcplex1260<y>worker.<ext> - Remote object worker library for protocol <y> 
	  libcplex1260<y>transport.<ext> - Remote object transport library for protocol <y> 
	  cplexamp - Solver driver to use with AMPL
	  convert - Conversion utility for files written in some older formats

  examples/src/<language>:
	Source files for the examples (lpex1.c is in examples/src/c, et cetera).

  examples/<machine>/<libformat>:
	Makefile for examples to support your particular platform.

  include/ilcplex:
	Files, such as cplex.h and ilocplex.h, that are needed to compile applications
	that use the Callable Library or the C++ version of Concert Technology.

  lib:
	The cplex.jar file containing the Java classes needed to use the Java version
	of Concert Technology.

  lib/<machine>/<libformat>:
	Library files that contain CPLEX (libcplex.a) and CPLEX extensions to Concert
	Technology (libilocplex.a).

  matlab:
	Files comprising the CPLEX for MATLAB interface.  
	Refer to the section "Setting up CPLEX for MATLAB" of "Getting started with CPLEX for MATLAB"
	for more information on configuiring your environment.

  python/<machine>:
	Files comprising the Python interface for CPLEX.  A setup.py script which
	runs distutils is included.

In the concert directory:

  include/ilconcert:
	Files, such as ilomodel.h, that are needed to compile applications that use the
	C++ version of Concert Technology.

  lib/<machine>/<libformat>:
	Concert Technology libraries for C++ users (libconcert.a).

In the doc directory:

      Documentation for the CPLEX Optimizers as well as the other components of CPLEX Optimization Studio.

Compiling and Executing

To compile and execute examples, do the following:
	% cd examples/<machine>/<libformat>
	% make lpex1     #  to compile and execute the first CPLEX example

A list of all the examples that you can run this way is to be found in the file Makefile by looking for C_EX (C examples), CPP_EX (C++ examples), JAVA_EX (Java examples), or you can view the files listed in examples/src.

You can also do
	% make all		#  to compile all examples
	% make execute		#  to compile and execute all examples
	% make execute_c	#  to compile and execute only the C examples
	% make execute_cpp	#  to compile and execute only the C++ examples
	% make execute_java	#  to compile and execute only the Java examples
	% make clean		#  to remove executable files when you are done

Many of the examples can be run in more than one way; the make execute command runs each in a representative way.

The file Makefile contains recommended compiler flags and other settings for building applications on your particular computer. You can find these settings in the sections of Makefile marked by the comments "Compiler options" and "Link options". The Makefile also shows how to set the location of the CPLEX shared library and cplex.jar file used from Java.
